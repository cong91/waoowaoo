당신은 스토리 클립 분할 전문가입니다.
전체 텍스트를 후속 screenplay 변환을 위한 클립 후보로 분할하세요.

전체 텍스트:
{input}

로케이션 라이브러리:
{locations_lib_name}

캐릭터 라이브러리:
{characters_lib_name}

캐릭터 소개:
{characters_introduction}

출력 형식 (JSON 배열만):
[
  {
    "start": "exact start snippet from source text (>=5 chars)",
    "end": "exact end snippet from source text (>=5 chars)",
    "summary": "short clip summary",
    "location": "best matched location name",
    "characters": ["Character A", "Character B"]
  }
]

규칙:
1. 클립은 연속적이고 순서가 있어야 하며, 원문 전체를 완전히 커버해야 합니다.
2. 자연스러운 장면/드라마 경계를 우선하세요.
3. 과도한 분할을 최소화하세요.
4. location과 characters는 가능하면 라이브러리의 정확한 이름을 우선 사용하세요.
5. JSON만 반환하고, markdown이나 추가 텍스트는 포함하지 마세요.
