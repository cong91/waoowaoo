# Task 5 — Clean production workspace project list to improve trust

## Problem
Production workspace currently exposes internal/test-style project names that reduce trust and perceived polish.

## Goal
Reduce visible internal noise in production-facing project lists.

## Acceptance criteria
- internal/test artifacts do not dominate the default project list view for real users
- project list feels customer-facing and intentional
- if internal/test projects must remain visible for operators, they are clearly separated or filtered

## Examples from audit
- `UX Manga Flow Test ...`
- `VAT132 Story Text Run ...`
- placeholder names such as `fasda`, `1231`
