# Task 2 — Rewrite beginner manga entry copy and primary CTA

## Problem
Current beginner-facing copy is capability-oriented and the primary CTA wording does not match user expectation.

## Current examples
- workspace card CTA: `New Project`
- workspace description: `Build manga/webtoon scenes with chapter continuity and panel-first controls`
- main generation CTA: `Tạo Script Demo (Manga)`

## Goal
Make the entry path speak in user-task language:
- start from an idea
- choose a style
- generate manga panels

## Acceptance criteria
- workspace manga card uses beginner-friendly copy
- primary CTA describes the next user-visible outcome
- helper line clearly explains minimum starting input (for example: 1–3 sentences)
- beginner user can infer the next step without understanding internal concepts like style lock / chapter continuity / runtime lane

## Suggested direction
- `Start Manga from Idea`
- `Create Manga Project`
- `Generate Manga Panels`
- helper text such as `Start with 1–3 sentences. No screenplay required.`
